Grid布局

1、grid基础介绍

1.1 grid容器

当给标签设置display: grid时，就形成了一个网格容器。

1.2 grid项目

grid容器的直接后代元素就是grid项目，例如：

1.3 网格线

构成网状结构的分界线。

1.4 单元格

网状线中间的单元区域。

1.5 网格轨道

两条相邻网格线之间的空间。即某行或某列

1.6 网格间隙

网格轨道之间的间隙

2、grid相关属性

2.1 display

属性值：

  • **grid**：块状网格
  • **inline-grid**：行内网格

2.2 grid-template-columns、grid-template-rows

用于定义列的列宽和行的行高

2.3 grid-template-areas

通过引用 **grid-area** 属性指定的网格区域的名称，定义一个网格模板。重复一个网格区域的名称会使内容跨越这些单元格。一个句号表示一个空单元。语法本身提供了一个可视化的网格结构。

属性值：

  • <grid-area-name>：用grid-area指定的一个网格区域的名称。
  • .：句号表示一个空的网格单元
  • none：没有定义网格区域

这将创建一个四列宽、三行高的网格。整个顶行将由页眉区域组成。中间一行将由两个主要区域、一个空单元和一个侧边栏区域组成。最后一行是所有的页脚。如图所示：

2.4 grid-template

是grid-template-rows+grid-template-columns+grid-template-areas的简写形式

2.5 column-gap、row-gap

列间隙和行间隙

2.6 gap

row-gap和column-gap的简写

2.7 justify-items

沿着内联（行）轴对齐网格项目（与沿着块（列）轴对齐的 align-items 相反）。这个值适用于容器内的所有网格项目。

属性值：

  • start：使项目与单元格的起始边缘齐平。
  • end：对齐项目，使其与单元格的末端边缘齐平
  • center：使项目在其单元格的中心位置对齐
  • stretch：填充整个单元格的宽度（这是默认值）

2.8 align-items

与justify-items类似

属性值：

  • start：使项目与单元格的起始边缘齐平。
  • end：对齐项目，使其与单元格的末端边缘齐平
  • center：使项目在其单元格的中心位置对齐
  • stretch：填充整个单元格的宽度（这是默认值）
  • baseline：沿着文本基线对齐项目。基线有一些修饰词–第一条基线和最后一条基线，在多行文本的情况下，它们将使用第一行或最后一行的基线。

2.9 palce-items

align-items和justify-items的缩写

2.10 justify-content

沿内联（行）轴对齐网格（与沿块（列）轴对齐网格的 align-content相反）

属性值：

  • **start**：使网格与网格容器的起始边缘齐平。
  • **end**：使网格与网格容器的末端边缘齐平。
  • **center**：使网格在网格容器的中心对齐。
  • **stretch**：调整网格项目的大小，让网格填满网格容器的全部宽度
  • **space-around**：在每个网格项目之间放置一个均匀的空间，在远端有一半的空间。
  • **space-between**：在每个网格项之间放置一个均匀的空间，远端没有空间
  • **space-evenly**：在每个网格项目之间放置一个均匀的空间，包括远端。

2.11 align-content

与justify-content类似

2.12 place-conetnt

align-content和justify-content的缩写

2.13 grid-auto-columns、grid-auto-rows

grid-auto-columns 和 grid-auto-rows 用来指定任何自动生成的网格轨迹（又称隐式网格轨迹）的大小。当网格中的网格项多于单元格时，或者当一个网格项被放置在显式网格之外时，就会创建隐式轨道。

此时b所在行列数已经超出了grid-box所规定的行列数（显式网格），所以创建了宽度为0的隐式网格来填充网格。

可以使用 grid-auto-columns和 grid-auto-rows 来指定这些隐式轨道的宽度。

2.14 grid-auto-flow

如果没有明确放置在网格上的网格项目，自动放置算法会启动，自动放置这些项目。这个属性控制了自动摆放算法的工作方式。

属性值：

  • row：依次填入每一行，必要时添加新行（默认值）
  • column：依次填入每一列，必要时添加新列
  • dense：如果较小的项目出现在网格中，则尝试在网格中较早的地方填上洞。

3、grid项目相关属性

需要注意的是：float, display: inline-block, display: table-cell, vertical-align 和 column-* 属性对网格 items 没有影响。

span关键字

如果你用 grid-column 或 grid-row 把项目放到它们的父网格上，当项目应该跨越多于一列或一行时，你可以使用 span关键字 来避免指定结束线。

示例：

item跨越2-5列，1-3行，可以用span关键字规定至少跨越到行列数，避免指定结束线

3.1 grid-column-start、grid-column-end、grid-row-start 和 grid-row-end

指定一个item的位置

属性值：

  • ****：可以是一个数字，指的是一个有编号的网格线，或者是一个名称，指的是一个有名称的网格线
  • **span **：该项目将跨越所提供的网格行数
  • **span **：项目将跨越，直到它到达下一个具有所提供的名称的行。
  • auto：表示自动放置，自动跨度，或者默认跨度为1。

3.2 grid-column、grid-row

  • grid-row 属性是 grid-row-start 和 grid-row-end 的简写形式
  • grid-column 属性是 grid-column-start 和 grid-column-end 的简写形式

3.3 grid-area

给予一个项目一个名字，以便它可以被用 grid-template-areas 属性创建的模板所引用。另外，这个属性可以作为 grid-row-start + grid-column-start + grid-row-end + grid-column-end 的一个更简短的缩写。

属性值：

  • <name>：你选择的名称
  • <row-start> / <column-start> / <row-end> / <column-end>：可以是数字或命名的行。

3.4 justify-self

属性值：

  • start
  • end
  • center
  • stretch

3.5 align-self

与justify-self类似

3.6 place-self

align-self与justify-self的缩写
