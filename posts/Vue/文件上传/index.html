文件上传

相关知识点

文件传输协议

  1. application/json：适用于base64格式的文件传输
  2. multipart/form-data：以二进制流的形式传输文件

File

File对象是Blob对象的子类，通常从用户使用<input>元素选择文件返回的FileList对象或拖放操作返回的DataTransfer对象中检索

ArrayBuffer

ArrayBuffer对象用来表示通用的原始二进制数据缓冲区。

它是一个字节数组，不能直接操作，而是要通过类型化数组对象或DataView对象来操作。

FormData

FormData 接口提供了一种表示表单数据的键值对 key/value 的构造方式。

便于使用编码类型multipart/form-data

base64

Base64 是一种将二进制数据编码为 ASCII 字符串的方法。它通常用于在 URL、JSON 或 XML 中嵌入二进制数据。

可以使用 FileReader 对象将文件转换为 Base64 编码字符串。

Blob流

Blob表示一个不可变的、原始数据的类文件对象。Blob几乎支持所有文件类型

Blob的用途：

  • 分片上传
  • 从互联网下载数据
  • Blob用作URL
  • Blob转换为Base64
  • 图片压缩
  • 生成PDF

Blob用作URL

得到一个Blob对象，可以通过URL.createObjectURL()方法将blob转换为一个url

浏览器内部为每个通过 URL.createObjectURL 生成的 URL 存储了一个 URL → Blob 映射。因此，此类 URL 较短，但可以访问 Blob。生成的 URL 仅在当前文档打开的状态下才有效。它允许引用 <img>、<a> 中的 Blob，但如果你访问的 Blob URL 不再存在，则会从浏览器中收到 404 错误。

虽然存储了 URL → Blob 的映射，但 Blob 本身仍驻留在内存中，浏览器无法释放它。映射在文档卸载时自动清除，因此 Blob 对象随后被释放。

针对这个问题，我们可以调用 URL.revokeObjectURL(url) 方法，从内部映射中删除引用，从而允许删除 Blob（如果没有其他引用），并释放内存。

计算hash值

计算hash值需要占用大量进程时间，为了防止进程阻塞，使用Worker线程计算。

hash.ts：

  • Worker线程内部无法使用import和require，其内部提供了importScripts方法用于导入脚本
  • 使用spark-md5获得hash映射
  • Blob（File）对象只是保存了文件的基本信息（如类型，大小等），文件的内容数据需要使用FileReader读取
